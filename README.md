# 음성 인식 기반 치매 위험도 모니터링 시스템

## 1. 프로젝트 개요

본 프로젝트는 사용자의 **일상 음성 대화**를 입력받아 텍스트로 변환하고, 언어적 특징을 분석하여 **치매 위험도를 정량적으로 평가하는 AI 기반 모니터링 시스템**이다.  
기존 치매 진단은 병원 방문, 설문, 전문 검사 등 사후적이고 정적인 방식에 의존하는 경우가 많다. 반면 본 시스템은 일상적인 발화를 기반으로 인지 저하의 초기 징후를 탐지하여, **조기 선별 도구**로 활용될 수 있는 가능성을 제시한다.

이 시스템은 다음 기능을 하나의 흐름으로 통합한다.

- 음성 입력 및 녹음
- Speech-to-Text(STT) 변환
- 대화형 AI 응답 생성
- 언어 패턴 기반 위험도 분석
- Recall Memory Test를 통한 단기 기억 확인
- 점수, 추세, 특징 분포의 시각화

즉, 단순 챗봇이 아니라 **대화 데이터 기반 인지 상태 분석 및 모니터링 시스템**이라는 점이 본 프로젝트의 핵심이다.

---

## 2. 프로젝트 필요성

고령화 사회로의 진입과 함께 치매 환자 수는 지속적으로 증가하고 있으며, 조기 발견과 초기 대응의 중요성 또한 커지고 있다. 치매 초기에는 일상 대화에서 다음과 같은 언어적 특징이 나타날 수 있다.

- 같은 질문을 반복하는 현상
- 기억이 나지 않는다는 표현의 증가
- 시간, 날짜, 현재 상황에 대한 혼란
- 문장 응집성 저하 및 비논리적 표현

이러한 특징은 정식 의료 검사 이전 단계에서도 관찰될 수 있으므로, 음성 및 대화 데이터를 기반으로 분석하는 AI 시스템은 **비침습적이며 접근성이 높은 선별 도구**로서 의미를 가진다.  
본 프로젝트는 이러한 문제의식에서 출발하여, 사용자의 발화를 분석하고 그 결과를 수치와 시각 자료로 제공함으로써 **직관적이고 연속적인 인지 상태 모니터링**을 지원하고자 한다.

---

## 3. 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

1. 사용자의 음성을 안정적으로 입력받아 텍스트로 변환하는 기능 구현
2. 대화형 AI를 통해 간단하고 직접적인 응답 제공
3. 대화 내용 속 언어적 특징을 분석하여 치매 위험도 점수 산출
4. 위험도 점수와 추세를 시각적으로 표현하는 대시보드 구현
5. Recall Memory Test를 통해 단기 기억 확인 요소 추가
6. 세션 단위 누적 분석을 통해 일회성이 아닌 모니터링 구조 구현

---

## 4. 시스템 전체 구조

본 시스템은 Flask 기반 웹 애플리케이션으로 구현되었으며, 백엔드와 프론트엔드가 다음과 같이 연동된다.

```text
사용자
  ↓
브라우저(녹음 / UI)
  ↓
Flask 서버
  ├─ 음성 파일 저장
  ├─ Google Cloud Speech-to-Text
  ├─ LLM 응답 생성
  ├─ 언어 특징 분석 및 점수 산출
  ├─ Recall Memory Test 상태 관리
  └─ 세션별 점수 이력 저장
  ↓
JSON 응답
  ↓
브라우저 대시보드
  ├─ 채팅 출력
  ├─ 위험도 카드
  ├─ 점수 추세 그래프
  ├─ 게이지 차트
  ├─ 레이더 차트
  └─ 경고 팝업
```

### 구성 요소

- **Frontend**
  - 음성 녹음 시작/중지
  - 사용자 발화 및 AI 응답 표시
  - 점수 카드, 추세 그래프, 레이더 차트, 게이지 차트 렌더링
  - 위험 점수 임계치 초과 시 경고 팝업 표시

- **Backend**
  - 업로드된 음성 파일 저장
  - Google STT 호출
  - 로컬 LLM 기반 응답 생성
  - 치매 위험도 분석 및 세부 점수 산출
  - 세션별 대화 기록, 점수 기록, Recall 상태 관리

- **AI / 분석 계층**
  - 질문에 짧게 답하는 응답용 LLM
  - 정해진 형식으로 위험도 점수를 산출하는 분석용 LLM
  - 출력 검증 및 재시도 로직

## 5. 세부 동작 구조

본 시스템은 다음 순서로 동작한다.

### 5.1 음성 입력 및 업로드

사용자는 웹 브라우저에서 마이크 권한을 허용한 뒤 음성을 녹음한다.  
프론트엔드는 `MediaRecorder`를 사용하여 음성 데이터를 수집하고, 녹음 종료 시 이를 `wav` 파일 형태로 서버에 전송한다.

### 5.2 음성 인식 단계

백엔드는 업로드된 음성 파일을 `uploads/` 폴더에 저장한 뒤, Google Cloud Speech-to-Text API를 호출하여 한국어 텍스트로 변환한다.

- 언어 코드: `ko-KR`
- 자동 문장부호 사용
- 다중 대안 중 최상위 결과 사용

변환된 텍스트는 불필요한 공백 및 노이즈 문자를 제거한 뒤 다음 단계로 전달된다.

### 5.3 대화형 AI 응답 생성

정제된 텍스트는 로컬 LLM에 입력되어, 사용자의 질문에 대해 짧고 직접적인 응답을 생성한다.  
이때 응답 생성 프롬프트는 다음 원칙을 따른다.

- 한국어만 사용
- 질문에 없는 사실은 추측하지 않음
- 상황, 장소, 인물 등을 임의로 추가하지 않음
- 2~4문장 이내의 간결한 응답 유지

즉, 응답용 모델은 대화를 길게 이어가는 역할보다, **간단한 상호작용 인터페이스** 역할을 수행한다.

### 5.4 언어 특징 분석 및 위험도 산출

동일한 사용자 입력은 분석용 LLM에도 전달된다. 분석용 LLM은 발화 내용에서 치매와 관련된 언어적 특징을 점수화한다.

분석 항목은 총 4가지이다.

1. **질문 반복**
2. **기억 혼란 표현**
3. **시간/상황 인식 혼란**
4. **문장 비논리성**

각 항목은 아래 범위 내에서 점수가 부여된다.

- 질문 반복: 0~25점
- 기억 혼란: 0~25점
- 시간 혼란: 0~30점
- 문장 비논리성: 0~20점

최종 위험도 점수는 네 항목의 합으로 계산되며, 총점 범위는 `0~100점`이다.

### 5.5 분석 결과 형식 검증

분석용 LLM은 반드시 정해진 형식으로 결과를 출력해야 한다. 그러나 LLM 출력은 형식이 흔들릴 수 있으므로, 본 시스템은 다음 보정 절차를 포함한다.

- 필수 항목 존재 여부 확인
- 각 점수가 정수인지 확인
- 총점이 세부 점수 합과 일치하는지 확인
- 분석 근거가 실제 문장 2문장 이상인지 확인
- 형식 오류 시 최대 3회 재시도
- 최종 실패 시 기본 fallback 결과 반환

이 과정은 단순한 생성형 응답이 아니라 **정형 데이터 기반 분석 결과**를 안정적으로 확보하기 위한 장치이다.

### 5.6 Recall Memory Test

일반 대화 분석만으로는 단기 기억 저하를 직접 반영하기 어렵기 때문에, 본 시스템은 보조 기능으로 Recall Memory Test를 포함한다.

동작 방식은 다음과 같다.

1. 일정 대화 턴 수에 도달하면 시스템이 하나의 단어를 제시
2. 사용자는 해당 단어를 잠시 기억
3. 다음 대화 차례에서 시스템이 단어를 기억하는지 질문
4. 사용자의 답변에 목표 단어가 포함되어 있는지 판별
5. 정답/오답 결과를 상태 카드에 반영

이 기능을 통해 언어 패턴 분석 외에 **단기 기억 확인 요소**를 시스템에 포함하였다.

### 5.7 세션 단위 이력 관리

시스템은 사용자별 세션 ID를 생성하여 아래 정보를 메모리 상에서 관리한다.

- 대화 기록
- 위험도 점수 기록
- Recall Test 진행 상태

세션이 유지되는 동안 최근 대화와 점수 이력이 누적되며, 이를 기반으로 다음 정보가 계산된다.

- 전체 평균 점수
- 최근 5회 평균 점수
- 최근 점수
- 점수 추세 상승/하락/유지

### 5.8 시각화 및 경고 처리

프론트엔드는 서버 응답을 기반으로 다음 시각 요소를 갱신한다.

- **상태 카드**: 최근 평균 점수 기준 위험 단계 표시
- **요약 카드**: 전체 평균, 최근 평균, 최근 점수, 추세
- **라인 차트**: 시간대별 점수 변화
- **게이지 차트**: 최근 평균 점수의 직관적 표시
- **레이더 차트**: 언어 특징 4개 점수 분포
- **Recall 카드**: 기억 테스트 상태 및 최근 결과

또한 현재 점수 또는 최근 평균 점수가 일정 기준 이상이면, 사용자에게 위험 경고 팝업을 표시한다.

## 6. 분석 기준

본 시스템은 아래 기준에 따라 위험도를 산출한다.

| 분석 항목 | 설명 | 점수 범위 |
|----------|------|-----------|
| 질문 반복 | 같은 질문 또는 유사 의미 질문 반복 여부 | 0 ~ 25 |
| 기억 혼란 | "기억이 안 난다", "뭐였지" 등의 표현 | 0 ~ 25 |
| 시간/상황 혼란 | 날짜, 시간, 현재 상황 인식 오류 | 0 ~ 30 |
| 문장 비논리성 | 문장 구조 불안정, 응집성 저하 | 0 ~ 20 |

### 종합 점수 기준

| 점수 구간 | 위험도 해석 |
|----------|-------------|
| 0 ~ 19 | 정상 |
| 20 ~ 39 | 약한 의심 |
| 40 ~ 59 | 중간 의심 |
| 60 ~ 79 | 높은 의심 |
| 80 ~ 100 | 매우 높은 의심 |

### 시스템 내부 위험 레벨

시각화 화면에서는 최근 평균 점수를 기준으로 다음 레벨을 표시한다.

- `Normal`
- `Low Risk`
- `Moderate Risk`
- `High Risk`
- `Very High Risk`

## 7. 주요 기능 요약

### 7.1 음성 인식 기능

- 브라우저에서 직접 녹음 가능
- Google STT를 활용한 한국어 음성 인식
- 입력 편의성을 높여 고령층 사용 가능성 고려

### 7.2 AI 대화 응답 기능

- 사용자 질문에 대해 짧고 명확한 응답 생성
- 불필요한 정보 확장 없이 핵심 응답 중심

### 7.3 치매 위험도 분석 기능

- 언어적 특징을 기반으로 점수화
- 세부 점수와 총점을 동시에 제공
- 판단 근거 텍스트 출력

### 7.4 Recall Memory Test

- 특정 단어 기억 여부를 확인하는 보조 인지 테스트
- 단기 기억 관련 지표를 대화 흐름 안에 포함

### 7.5 실시간 시각화 기능

- 카드, 선 그래프, 게이지, 레이더 차트 제공
- 점수 변화와 특징 분포를 직관적으로 확인 가능

### 7.6 기록 초기화 기능

- 세션별 누적 기록 초기화 가능
- 새로운 사용자 또는 새 평가 흐름 시작 가능

## 10. 실행 방법

### 10.1 환경 준비

```bash
pip install -r requirements.txt
```

### 10.2 필요 자원

본 프로젝트 실행을 위해서는 아래 자원이 필요하다.

- Google Cloud Speech-to-Text 서비스 계정 키 파일
- `models/` 폴더 내부 GGUF 형식 LLM 모델 파일

### 10.3 실행

```bash
python app.py
```

또는 Windows에서는 아래 배치 파일로 바로 실행할 수 있다.

```bash
start_server.bat
```

서버 실행 시 콘솔에 다음 정보가 출력된다.

- Local URL: 현재 컴퓨터에서 접속하는 주소
- LAN URL: 같은 네트워크의 다른 컴퓨터에서 접속하는 주소

예를 들어 LAN URL이 아래처럼 출력되었다면,

```text
http://192.168.0.15:5000
```

같은 공유기 또는 같은 사설망에 연결된 다른 컴퓨터에서도 위 주소로 접속할 수 있다.

기본적으로 서버는 `0.0.0.0:5000` 에 바인딩되며, 환경 변수로 변경할 수 있다.

```bash
HOST=0.0.0.0
PORT=5000
WAITRESS_THREADS=8
ANALYSIS_N_CTX=8192
ANALYSIS_N_BATCH=512
ANALYSIS_MAX_TOKENS=384
```

분석용 LLM만 더 넓은 문맥과 배치를 사용하도록 분리했다. 답변용 모델은 기존 설정을 유지한다. 메모리 사용량이 부담되면 `ANALYSIS_N_CTX` 또는 `ANALYSIS_N_BATCH`를 낮춰 조절할 수 있다.

### 10.4 접속 주소

```text
http://localhost:5000
```

같은 네트워크의 다른 컴퓨터에서는 아래 형식으로 접속한다.

```text
http://서버컴퓨터_IP주소:5000
```

예:

```text
http://192.168.0.15:5000
```

### 10.5 외부 인터넷 공개 시 참고 사항

다른 장소의 컴퓨터에서 인터넷을 통해 접속하려면 추가 설정이 필요하다.

- 서버 PC의 방화벽에서 포트 허용
- 공유기 포트포워딩 설정
- 공인 IP 또는 도메인 연결
- HTTPS 적용 시 리버스 프록시(Nginx, Caddy 등) 또는 클라우드 배포 환경 사용

즉, 현재 구조는 **같은 네트워크 안에서는 바로 접속 가능한 형태**로 구성되며, 외부 공개형 서비스로 확장할 때도 운영 서버 위에 배포할 수 있도록 준비된 구조이다.

## 12. 한계점 및 개선 방향

본 프로젝트는 치매를 진단하는 의료 시스템이 아니라, **언어적 징후를 기반으로 위험도를 추정하는 보조 시스템**이다. 따라서 다음과 같은 한계가 존재한다.

- 실제 임상 데이터 기반 검증이 부족함
- 단일 세션 또는 짧은 대화만으로는 강한 판단이 어려움
- LLM 분석 결과가 프롬프트와 출력 형식에 영향을 받을 수 있음
- 의료적 진단을 대체할 수 없음

향후에는 다음 방향으로 개선할 수 있다.

- 실제 고령층 발화 데이터셋 기반 정량 평가
- 장기 세션 기록 저장 및 사용자별 변화 추적
- 감정 분석, 얼굴 표정, 행동 데이터와의 융합
- 전문가 피드백 기반 분석 기준 고도화

## 13. 결론

본 프로젝트는 음성 인식, 자연어 처리, 대화형 AI, 시각화 기술을 결합하여 **사용자의 일상 대화로부터 인지 저하 징후를 탐지하는 통합형 시스템**을 구현하였다.

본 시스템은 실제 의료 진단 도구를 대체하지는 않지만, 치매 조기 선별 및 일상 기반 모니터링이라는 측면에서 충분한 연구적·실용적 의의를 가진다.
